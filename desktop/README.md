# Schwab Trader — Desktop (Electron + PyInstaller)

Packages the app as a **standalone Windows desktop app**: an Electron window that
loads the SPA served by a bundled Python backend exe. No Python, Node, Postgres, or
Docker on the target machine — the app carries a **PyInstaller** backend + **SQLite**
DB (created per-user on first run).

## Architecture
- **Backend sidecar** — `../backend/dist/schwab-backend/schwab-backend.exe`
  (PyInstaller, onedir). Electron spawns it with:
  - `SCHWAB_DATA_DIR` = the OS userData dir → SQLite DB + encrypted tokens live there
    (writable even when the app is installed under Program Files).
  - `SCHWAB_FRONTEND_DIR` = the bundled frontend `dist` → the backend serves the SPA
    **same-origin**, so no CORS.
  - `SCHWAB_PORT` = 8000 (the frontend's API base is `http://localhost:8000`).
- **Window** loads `http://localhost:8000/` once the backend answers `/api/quotes`.
- **First run**: no creds/token → the app opens in DEMO. The user enters their own
  Schwab developer-app creds (Settings → Schwab API credentials), then connects each
  profile. Everything persists in the per-user data dir.

## Build a local installer
```
# 1) build the two payloads
npm run prep          # frontend  -> ../frontend/dist
                      # backend   -> ../backend/dist/schwab-backend/ (PyInstaller)
# 2) install electron tooling (first time only)
npm install
# 3) produce the NSIS installer -> dist/
npm run dist
```

`npm run build:backend` runs `pyinstaller --noconfirm schwab-backend.spec` in
`../backend` (the spec was generated by the initial build; it bundles the strategy
YAML + the alembic migrations and collects uvicorn/schwab/alembic/aiosqlite).

## Releases + auto-update
`electron-updater` checks **GitHub Releases** on launch and installs updates on quit —
so shipping a new version is: bump `version`, tag, and publish.
1. In `package.json` → `build.publish`, set `owner`/`repo` to your GitHub repo.
2. `set GH_TOKEN=<a token with repo scope>` then `npm run release` — builds + uploads
   the installer + `latest.yml` to a GitHub Release.
3. Users on the old version auto-update on next launch. DB schema changes ride along:
   the backend runs `create_all` + `alembic stamp head` (SQLite) on boot.

## Notes / TODO before shipping widely
- **App icon**: add `build.win.icon` (a `.ico`) — currently the Electron default.
- **Code signing** (optional for a small trusted circle): unsigned installs show a
  one-time Windows SmartScreen "More info → Run anyway"; auto-update still works after.
  For wider distribution add a Windows signing cert (`build.win.certificateFile`).
- **Fixed port 8000**: if it's taken, startup waits then fails. A future refinement is a
  dynamic free port injected into the frontend's API base (currently hardcoded).
- **Future SQLite migrations must be batch-safe** (SQLite can't replay the historical
  Postgres-only ALTER/constraint migrations; the app builds SQLite from the models).
