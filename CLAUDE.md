## What this is
A local, always-on day-trading app for the **Schwab Trader API**, replacing a sophisticated Google Sheet ("Christian - Stock Trading for 2026"). Single user (Christian). Goals: never depend on Google; never visit Schwab's website; human-in-the-loop one-click trading (app does all API+DB work, user approves via buttons).

## Strategy — LIFO progressive ladder (config-driven, NOT fixed)
- Buy in rungs as price drops, **increasing size deeper**; sell last-in (cheapest) lots first; aim to ~double balance/year.
- **Sizing** by rungs already filled for the ticker: 1–2→$500, 3–7→$1000, 8–10→$1500 (shares = $ ÷ price).
- **Buy-sug ladder**: rung2 = prior×(1−10%); rungs3–7 = prior×(1−13%); rungs8–10 = prior×(1−16%).
- **Sell targets** per lot: flat-$ gain OR %-above-buy (both supported).
- Guardrails: <5%/stock, ~6 lots deep, US $1–30B, no China/biotech, rarely short.
- All rules live in `backend/app/strategy/default_strategy.yaml` + pure `rules.py`. **Never hardcode strategy in plumbing.**

## Stack / architecture
- Backend: Python 3.14 + FastAPI + `schwab-py` + async SQLAlchemy on psycopg3.
- DB: Postgres in Docker, host port **5433** (`docker compose up -d`).
- Frontend: React + Vite + TS; lightweight-charts (charts are opt-in).
- **Design system (use it — don't reintroduce drift):** `src/tokens.css` = the only home for color/type/space/radius/elevation, as CSS vars (semantic money pair: `--pos`/`--neg` for numbers, `--pos-strong`/`--neg-strong` for fills; `--accent` blue = a *control*, green = an *outcome*; text tiers `--text`/`-muted`/`-dim`/`-faint`, all WCAG-AA). `src/ui.css` = primitive classes (`.btn[-primary/secondary/ghost/buy/sell/danger/sm]`, `.field`, `.navtab`, `.tbl`, `.rowlink`+`.row-buy/-sell`, `.hover-reveal`, `.chip[-buy/-sell]`, `.panel`, `.page-title`/`.section-title`, `.modal-overlay`/`.modal`, `.toast`, `.skeleton`) — these carry the `:hover`/`:focus-visible`/media-query states inline styles can't. Components use classes + tokenized inline styles. Errors go through `useToast()` (`Toast.tsx`), never `alert()`. Money modals (OrderTicket/Reauth) are `role=dialog` + Escape + focus-trap. PriceChart is the ONE exception that must use literal hex (canvas can't read `var()`) — keep them aligned to token values.
- Live quotes: Schwab streamer → in-memory hub → `/ws/quotes`, `/ws/dashboard`.

## Run it
- Backend: `cd backend && .venv\Scripts\python.exe run.py` → http://localhost:8000  *(NOT bare uvicorn — Windows needs SelectorEventLoop; run.py sets it)*
- Frontend: `cd frontend && npm run dev` → http://localhost:5173
- One-shot: `./start.ps1` · stop: `./stop.ps1`
- **Ship a build: `./build-installer.ps1`** (reads `backend/app/version.py`, syncs package.json, vite→PyInstaller→electron-builder, prints the installer path). Bump `APP_VERSION` first when releasing.
- Re-auth Schwab (token lasts ~7 days): `cd backend && .venv\Scripts\python.exe -m app.schwab.authorize` (the app nags via an AuthBanner at ≤2 days / expired).
- DB schema is **Alembic-managed** — startup auto-runs `alembic upgrade head`. To change schema: edit `app/db/models.py` → `cd backend && .venv\Scripts\python.exe -m alembic revision --autogenerate -m "msg"` → review the file → `alembic upgrade head`. NEVER drop/recreate (you'd lose trade data).

## Gotchas (don't rediscover)
- **Fills aren't only `assetType == "EQUITY"`.** Schwab classifies some tradeable ETFs/funds as `COLLECTIVE_INVESTMENT` (also `ETF`, `MUTUAL_FUND`). The fill parser (`fills.py`) + `positions_sync.py` originally accepted EQUITY-only and SILENTLY DROPPED everything else — a real QBTX (COLLECTIVE_INVESTMENT) buy never became a lot / never hit the dashboard (2026-06-30). Fix: accept all SHARE-based instruments; only skip `_SKIP_ASSET_TYPES = {OPTION, FUTURE, FOREX}` (not share-based), and LOG the skip so it's never silently invisible. Options/futures would need separate (non-share) handling.
- Windows + psycopg async → must use SelectorEventLoop → always launch via `run.py`.
- Schwab **market-data API throttles bursts** → returns HTTP 200 with EMPTY candles (not 429). Mitigated by caching + a single shared client. Don't hammer it.
- **One shared schwab client** (`schwab/auth.py: get_client()`) — multiple clients race on the token file and fail intermittently.
- **Trading safety (server-side in orders.py, all verified):** orders only on the **selected** account and only if it's **trading_enabled** (per-account toggle, defaults OFF — explicit opt-in in Settings, never auto-enabled by type); a client `account_hash` may not diverge from the selected account; SELL **fails CLOSED** (refused unless held shares confirmed ≥ qty); STOP/STOP_LIMIT refused if the stop is the wrong side of the last price (immediate-trigger guard); all order types supported (market/limit/stop/stop-limit/trailing) — market/triggered fill at prevailing price.
- Only cancel orders **not** in a terminal status (denylist `_TERMINAL`); canceling REJECTED/FILLED returns HTTP 400.
- Deferred (low): no "limit price far from market" fat-finger soft-confirm yet — server accepts any positive limit. Add when …719 is funded.
- The original sheet had real bugs we did NOT copy (flat-12% tax cascade, month buckets that drift, "Gross Sales" col is actually cost basis, a phantom 366th day). App computes correctly + shows sheet-parity alongside.
